    Integers

    Integers are whole numbers (no decimal points) that are positive or negative. In other languages, there is also unsigned integers (only positive numbers), but Java does not have this. By design, int stores numbers from -2^31 to 2^31-1. This range isn't important to remember, but it is important to recognize its limit and when to use another datatype like long instead. If you are working with big numbers, use long to be safe 
    
int variable1 = 5;
int variableWithMultipleWords = -200;
int unassignedInt; //default value is 0.  
long bigNumber = 2562345234; //would not work on int

    In the last example, we see that 2562345234 is larger than 2^31-1, so an integer is unable to represent it, so we have to use long. What actually happens when we do int though? We would experience something called integer overflow, which means it loops back around to the negatives instead because of how binary works.

    0b11111110 = 254
    0b11111111 = 255
    0b100000000 = 256? Cant represent the 9th bit, so its ignored
    0b00000000 = 0

    In this example, we loop back to 0 since we were counting only in the positives, but in reality, this is offset by half the maximum value (128 in this case), so it would go from 127 to -128. Why the offset by 1? This is since the 0 also needs to be represented, giving us an odd number of values, and we would rather have a binary 0b10000000 be 0 rather than 0b10000001 be 0.

    Decimal numbers (Float/Double)

    With integers, representing them in binary is simple. However, decimal numbers is much harder. How do we represent the decimal and whole number portion? What about really big numbers? What about really small numbers? Well, we don't need to worry about that for the AP Exam, but you should be aware that Floats/Doubles do this with some clever math implementation with fractions and exponents that does have some form of inaccuracy called point float error. Since we are still limited to 32 or 64 bits, we can not represent every possible number, since it would be infinite, so we just use a "good enough" representation with a few digits of accuracy. In the exam, we don't care about optimizing, so we never really need to use float, we can stick with double only. Doubles have more accuracy and a much greater range, so we are less likely to run into any limitations for it, whereas floats have a lower accuracy and range. The benefit of using a float over a double is just memory usage and compatability with very old systems.

      if - Used to start an if-statement
      else if - Used to run only if the precding statement is false
      else - Runs only if everything is false

    Nesting Loops

    Neting loops is when we put a loop inside another loop — typically when we are working with 2D arrays. However, nesting loops is not only used for those purposes Sometimes when trying to solve a problem involving a single number, we may need to use a loop to solve that problem. An example would be counting in binary without using any conversion. Heres how you would count up to 1 in binary: 

    
for(int i=0 ; i < 2  ;i++){
    System.out.println(i);
}

    But wheres the nesting in this function? I just used 1 underwhelming for-loop here. What if we try to count to 3 using this, how do we do it? 

    
for(int j=0 ; k < 2 ; j++){
    for(int i=0 ; i < 2  ;i++){
        System.out.print(j);
        System.out.println(i);
    }
}

    So how does that work? When we nest for-loops for each value j (in our exammple above), i has to go through 0 and 1 before j can increase to 1. After it increases to 1, we re-run the code inside the j for-loop, which runs the i for-loop again, going through i=0 and i=1 again.
    Going through it step by step, j starts at 0, and so does i. We then print the value of j, then the value of i, followed by a newline

    j=0, i=0

    Next, we're still in the i for-loop since we havent finished running it yet. Now i gets incremented by 1, and when we run the prints, we get:
    j=0, i=1

    Now when i=1, it gets incremented to 2, which then causes the for-loop to stop. After the i for-loop ends, we also finish the j for-loop, so j gets incremented by 1, and restart the i for-loop starting at 0
    j=1, i=0

    We then repeat the process after the print statements, i gets incremented and repeats
    j=1, i=1

    Now when i increments to 2, the i for-loop stops, which completes an iteration of the j for-loop, incrementing j to 2, which then stops the j-for loop. So thats how this works.
    This would work counting in octal (base 8: 0,1,2,3,4,5,6,7,10,11,12,13,14,15,16,17,20) that no one uses, as well as hex with a few modifications. If you want to be able to print more digits, you can add more for-loops. This is not the most efficient way of counting in another base, but it is good for introducing the concept of nesting loops and how it works.

    In essence, any loop inside another loop must finish executing before proceeding. In a way, it runs similar to the code below: 
    
int j=0;
for(int i=0 ; i < 2  ;i++){
    System.out.print(j);
    System.out.println(i);
}
